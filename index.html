Sistema de Reserva de Aulas

Documentación y diseño del sistema para la universidad

Introducción

Bienvenido a la documentación del nuevo Sistema de Reserva de Aulas. Este sistema ha sido diseñado para facilitar a los estudiantes la gestión y solicitud de espacios físicos dentro de la universidad para realizar proyectos, reuniones o tutorías. A continuación, se detallan los diagramas principales que explican la arquitectura y el flujo de trabajo del sistema.

1. Diagrama de Casos de Uso

Este diagrama muestra las interacciones principales entre los usuarios (actores) y el sistema. En este modelo identificamos a dos actores clave:

  • Cuenta Habiente: Representa al usuario general del sistema. Puede registrarse, consultar la disponibilidad de las aulas, realizar una reserva y cancelar una reserva si ya no es necesaria.
  • Técnico: Es el encargado de administrar y gestionar las peticiones, teniendo la facultad de aprobar o rechazar las solicitudes de reserva realizadas por los cuentas habientes.

2. Diagrama de Actividades

El diagrama de actividades detalla el flujo de los procesos paso a paso, dividiendo las responsabilidades en tres carriles: Alumno, Sistema y Administrador del Sistema.

El flujo principal comienza cuando el alumno verifica si tiene cuenta, procede a registrarse o iniciar sesión y accede al menú. A partir de ahí, puede:

  • Hacer una reserva: El sistema le muestra las aulas y horarios, el alumno escoge, y la solicitud pasa al administrador. El administrador consulta las solicitudes y decide si aprueba o no, desencadenando notificaciones.
  • Cancelar reserva: El sistema muestra las reservas activas, el alumno escoge cuál cancelar, y el sistema actualiza la base de datos de manera directa.

3. Diagrama Entidad-Relación

Este diagrama modela la estructura de la base de datos necesaria para soportar el sistema de reservas. Se compone de cuatro entidades principales:

  • ALUMNO: Almacena la información de los estudiantes (ID, nombre, correo institucional único y contraseña).
  • AULA: Contiene el catálogo de espacios de la universidad (ID, nombre/número y capacidad).
  • ADMINISTRADOR: Guarda los accesos del personal encargado de gestionar el sistema (ID, nombre, correo y contraseña).
  • RESERVA: Es la entidad transaccional que une al alumno, al aula y al administrador. Registra las fechas, horas, el estado de la solicitud y el motivo del uso.
